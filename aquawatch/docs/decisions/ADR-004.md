# ADR-004: Rule-Based AI Chatbot

## Status
**Accepted** - Currently in use

## Date
2024-01-20 (Feature Implementation)

## Context
We wanted to provide users with instant answers about water safety, hygiene, and sanitation topics without requiring a backend AI service. The chatbot should:
- Answer common questions about WASH topics
- Provide helpful tips without external API calls
- Work offline (after initial page load)

## Decision
Implement a rule-based chatbot using:
- **Keyword matching**: Detect user intent from message
- **Pre-defined responses**: Curated answers for each category
- **Random selection**: Vary responses to feel more natural

## Response Categories

| Category | Keywords | Example Responses |
|----------|----------|-------------------|
| `greeting` | hi, hello, hey | "Hello! I'm your WASH assistant..." |
| `water_safety` | brown, cloudy, smell, drink | "Boil water if it looks cloudy..." |
| `contamination` | contaminat, toxic, chemical | "Brown or discolored water..." |
| `hygiene` | wash hand, soap, disinfect | "Wash hands with soap for 20+ seconds..." |
| `sanitation` | toilet, sewage, drain | "Open drainage is a health hazard..." |
| `health` | sick, diarrhea, disease | "Diarrhea from contaminated water..." |
| `emergency` | urgent, flood, dead animal | "If you see dead animals..." |
| `general` | what is wash, explain | "WASH = Water, Sanitation..." |
| `water_sources` | cauvery, borewell, tanker | "Bengaluru gets water from..." |

## Implementation Pattern

```javascript
function getAIResponse(message) {
    const lowerMsg = message.toLowerCase();
    
    // Pattern 1: Check greeting
    if (lowerMsg.match(/^(hi|hello|hey|namaste)/)) {
        return AI_RESPONSES.greeting[random];
    }
    
    // Pattern 2: Complex matching
    if (lowerMsg.match(/brown|cloudy|smell/) && lowerMsg.match(/water|drink/)) {
        return AI_RESPONSES.water_safety[random];
    }
    
    // Default fallback
    return "I'm here to help with water safety...";
}
```

## Alternatives Considered

### Option 1: OpenAI API (GPT)
- **Pros**: Natural conversations, learns from context
- **Cons**: Requires API key, costs money, needs internet
- **Rejected**: Budget and simplicity concerns

### Option 2: Hugging Face Inference API
- **Pros**: Free tier available, good models
- **Cons**: Rate limits, requires setup
- **Rejected**: Still needs internet, not fully offline

### Option 3: Local AI (TensorFlow.js)
- **Pros**: Runs in browser, offline capable
- **Cons**: Large bundle size, slower
- **Rejected**: Too heavy for simple Q&A

### Option 4: Dialogflow
- **Pros**: Powerful NLP, visual builder
- **Cons**: Complex setup, Google Cloud required
- **Rejected**: Overkill for simple FAQ

## Consequences

### Positive
- ✅ Zero cost - no API calls needed
- ✅ Works offline (after page loads)
- ✅ Fast responses - no network latency
- ✅ Predictable, controlled responses
- ✅ Easy to customize responses
- ✅ No privacy concerns (data stays local)

### Negative
- ❌ Can't understand complex questions
- ❌ No conversation memory
- ❌ Limited to predefined topics
- ❌ Feels robotic vs. AI
- ❌ Maintenance burden as topics grow

## Future Considerations

For production, consider:
- **RAG (Retrieval Augmented Generation)**: Use embeddings to find relevant docs
- **GPT-4 API**: For natural conversations
- **Custom GPT**: OpenAI's fine-tuned option
- **Watson Assistant**: IBM's enterprise solution
- **Knowledge base expansion**: Add more response categories
